# ADR-0001: Initial Architecture & Technology Stack

## Status
Accepted

## Context
Нам необходимо выбрать базовый технологический стек для платформы декомпиляции и аудита APK. Платформа должна поддерживать тяжелые фоновые задачи (декомпиляция, запуск эмуляторов), обеспечивать высокую безопасность и быть расширяемой для добавления новых инструментов анализа.

## Decision Drivers
*   **Легкость интеграции с утилитами ИБ**: Большинство инструментов (Frida, Semgrep wrapper, Androguard) написаны на Python или имеют отличные биндинги к нему.
*   **Изоляция**: Необходима надежная, программно-управляемая изоляция "Air-Gap" для защиты хост-системы.
*   **Асинхронность**: Задачи анализа могут занимать от нескольких минут до часа.

## Considered Options

### Backend / Orchestration
1. **Python (FastAPI + Celery)**
   *   *Pros*: Нативная интеграция с ИБ-тулзами, огромная экосистема для парсинга и анализа, высокая скорость разработки.
   *   *Cons*: Производительность Celery при экстремально высоких нагрукзах ниже, чем у Go.
2. **Go**
   *   *Pros*: Высокая производительность, отличная конкурентность, статический бинарник.
   *   *Cons*: Придется делать wrapper-ы над Python ИБ-утилитами (вызов через CLI), что усложнит обработку ошибок.

### Sandbox
1. **Docker-Android**
   *   *Pros*: Легковесно, можно разворачивать программно через Docker API, легко интегрировать `iptables` для перехвата сети.
   *   *Cons*: Требует nested virtualization (KVM) на хосте.
2. **Виртуальные машины (QEMU/VirtualBox)**
   *   *Pros*: Максимальная изоляция.
   *   *Cons*: Тяжелые, долго запускаются, сложно автоматизировать провижининг.

## Decision
Мы будем использовать **Python (FastAPI + Celery)** в качестве бэкенда и оркестратора, PostgreSQL для метаданных, и **Docker-Android** в качестве песочницы с принудительной сетевой изоляцией (`mitmproxy`).

## Rationale
Выбор Python диктуется доменной областью (Кибербезопасность), где он является стандартом де-факто. Скорость интеграции с Frida и Semgrep важнее микросекундной производительности маршрутизации. Docker-Android дает идеальный баланс между скоростью запуска эфемерного окружения и уровнем изоляции для анализа мобильных приложений.

## Consequences
### Positive
*   Быстрый старт разработки.
*   Простое написание кастомных правил анализа на Python.

### Negative
*   Необходимость жестко контролировать зависимости Python.
*   Требование к хост-серверу: поддержка KVM (nested virtualization) для Docker-Android.
