# ADR-0002: Local LLM Integration (Ollama)

## Status
Accepted

## Context
В процессе автоматизированного аудита безопасности (SAST/DAST) генерируется большое количество "сырых" данных о потенциальных уязвимостях. Для фильтрации ложных срабатываний (False Positives) и предоставления аналитикам понятных рекомендаций по устранению, необходим слой ИИ-аналитики. Из соображений безопасности и соблюдения парадигмы "Zero-Trust Sandbox" использование облачных API (OpenAI, Anthropic) недопустимо, так как исходный код приложения не должен покидать контур системы.

## Decision Drivers
*   **Безопасность и Конфиденциальность**: Исходный код нельзя передавать сторонним провайдерам.
*   **Качество анализа кода**: Модель должна хорошо понимать Java/Kotlin и специфику Android.
*   **Стоимость и Автономия**: Бесплатная локальная инференция.

## Considered Options

### Облачные LLM (OpenAI GPT-4, Claude)
*   *Pros*: Лучшее качество "из коробки", не требуют ресурсов локальной машины.
*   *Cons*: Нарушение конфиденциальности (отправка декомпилированного проприетарного кода на внешние серверы). Отклонено по соображениям ИБ.

### Локальные LLM через Ollama
*   *Pros*: Полный контроль данных, высокая производительность на современных чипах (Apple Silicon / RTX), простота развертывания через REST API (`http://localhost:11434`).
*   *Cons*: Качество анализа зависит от доступного "железа" (RAM/VRAM) и выбора модели.

### Выбор модели для Ollama
Рассматривались модели: `qwen3.6:latest`, `deepseek-r1:14b`, `gemma3:12b`, `gpt-oss:20b`.
Выбор пользователя: Приоритет отдается `qwen3.6:latest` (отличные способности в кодинге и анализе).

## Decision
Интегрировать **Ollama** как локальный AI-движок для анализа уязвимостей. 
*   **Модель по умолчанию**: `qwen3.6:latest`.
*   Интеграция будет реализована на Backend (FastAPI) с использованием `httpx`.
*   Frontend будет предоставлять кнопку "Ask AI" / "Explain Risk" для каждого найденного инцидента.

## Consequences
### Positive
*   Абсолютная приватность исходного кода.
*   Возможность генерации интеллектуальных патчей и объяснений уязвимостей прямо в UI.
### Negative
*   Дополнительная нагрузка на хост-машину при генерации ответов.
*   Необходимость держать процесс Ollama запущенным на хосте (будет настроен проброс `host.docker.internal`).
