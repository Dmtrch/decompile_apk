# ADR-0003: Android SDK & Dockerized Emulator Integration

## Status
Accepted

## Context
Для обеспечения «замкнутого цикла» анализа (Build-Back Identity и Dynamic Sandbox Testing) системе требуется реальное сборочное окружение (Android SDK) и среда исполнения (Android Emulator). Ранее использовались моки (заглушки) и эмуляция процессов. Для получения достоверных данных аудита необходимо запускать декомпилированные и пересобранные APK на реальном устройстве с поддержкой ADB и Frida.

## Decision Drivers
*   **Достоверность анализа**: Результаты динамического анализа должны соответствовать работе приложения на реальном устройстве.
*   **Автоматизация сборки**: Команда `gradle assembleDebug` должна успешно выполняться в контуре Docker.
*   **Изоляция**: Эмулятор должен быть частью внутренней сети Docker, защищенной «Air-Gap» сетевым контуром.

## Decision
1.  **Android SDK**: Установить Android Command-line Tools напрямую в образ `worker`. Принять все лицензии и установить необходимые платформы (android-33) и build-tools.
2.  **Эмулятор**: Добавить в `docker-compose.yml` сервис `android-emulator` на базе образа `budtmo/docker-android`.
3.  **ADB Integration**: Связать воркер и эмулятор через ADB по сети Docker (`adb connect android-emulator:5555`).

## Consequences
### Positive
*   Возможность реальной проверки работоспособности пересобранных приложений.
*   Реальный перехват сетевого трафика и системных вызовов через Frida на живом устройстве.
### Negative
*   Значительное увеличение размера Docker-образа `worker` (на ~2-3 ГБ).
*   Высокое потребление ресурсов процессора и оперативной памяти при работе эмулятора.
*   Зависимость от поддержки KVM/виртуализации на хост-машине.
